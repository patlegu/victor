# Demande de licence commerciale — Victor

Ce document explique comment obtenir une licence commerciale dérogatoire pour Victor,
distribué sous **GNU Affero General Public License v3.0 (AGPL-3.0)**.

---

## Pourquoi une licence dérogatoire ?

L'AGPL-3.0 impose que toute version modifiée de Victor — y compris lorsqu'il est
utilisé comme service réseau (SaaS, API interne) — soit redistribuée avec son
code source sous la même licence.

Certaines organisations dont les politiques internes sont incompatibles avec cette
obligation (confidentialité du code, secret industriel, contraintes contractuelles)
peuvent solliciter une **licence commerciale dérogatoire** qui lève cette obligation.

---

## Restrictions d'accès

Le détenteur des droits se réserve le droit de **refuser toute demande** émanant
d'entités dont les activités ne sont pas en adéquation avec les valeurs du projet.

Sont notamment concernés, sans que cette liste soit exhaustive :

- Secteur de la défense et de l'armement
- Industrie de surveillance de masse
- Entités faisant l'objet de sanctions internationales

---

## Processus de demande

1. **Ouvrir une issue** sur le dépôt GitHub avec le label `[Licensing Request]`
2. Décrire l'organisation, le cas d'usage envisagé et le périmètre de déploiement
3. Une réponse sera apportée dans un délai raisonnable

> Les demandes incomplètes ou sans description d'usage ne seront pas traitées.

---

## Ce qui est toujours libre

Quelle que soit votre situation, les usages suivants restent **libres et gratuits**
sous AGPL-3.0, sans aucune formalité :

- Usage personnel ou interne non commercial
- Recherche académique et universitaire
- Projets open source redistribués sous AGPL-3.0 ou licence compatible
- Audit de sécurité, tests, évaluation
